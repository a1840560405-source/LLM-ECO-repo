# 文档索引（Docs）

这份目录是给“快速浏览”的：先看短文，再决定要不要读白皮书/跑 demo。

## 建议阅读顺序

1) `docs/case-study-1pager.md`  
   一页讲清楚：这个仓库的 demo 在跑什么、为什么这样设计、有哪些边界。

2) `docs/faq.md`  
   集中回答几个绕不开的问题：中文文本怎么处理、泄漏怎么防、EV 校正为什么做、DID 结果怎么看。

3) `whitepaper/executive_summary.md`  
   站在白皮书视角的总结：文本工具怎么和计量框架对接。

4) `whitepaper/paper.md`  
   完整版白皮书。

## 关于 `docs/consulting/`

`docs/consulting/` 里放的是我写交付文档时常用的模板/草稿（service brief、proposal、SOW）。

- 你可以把它当成“如何把研究做成可交付工程”的写作练习。
- 如果你只是想复现 demo，不需要看这一部分。
