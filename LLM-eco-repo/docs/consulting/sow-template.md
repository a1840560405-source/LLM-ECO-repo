## 1. 项目目标（Objectives）

## 2. 工作范围（Scope of work）

### 2.1 工作包 A：诊断与口径对齐（1–2 周）
**内容**
- 需求澄清：研究问题、决策场景、关键假设
- 指标口径：时间窗口、频率、缺失与异常处理
- 合规与治理：数据分类、脱敏策略、访问控制、留痕要求

**交付物**
- A1：问题定义与口径文档（含数据字典）
- A2：风险与偏误清单（leakage/EV/漂移/样本选择）

**验收**
- 客户确认口径冻结（baseline）并签字/邮件确认

### 2.2 工作包 B：PoC 与初版评估（3–6 周）
**内容**
- 文本对齐：检索/匹配策略、抽检与误差分析
- 特征构建：关键词/语义特征；漂移与质量控制
- 偏误校正：EV 校正（含抽样标注方案）、泄漏防护
- 评估：DID/事件研究或双方确认的方法；敏感性分析

**交付物**
- B1：可复现 Demo（Notebook + script pipeline）
- B2：指标解释卡（feature cards）
- B3：初版结果汇报（含可视化与要点结论）

**验收**
- PoC 一键运行通过；输出核心表格/结论与版本号

### 2.3 工作包 C：工程化交付与内化（6–12 周，可选）
**内容**
- 工程化：数据输入/输出规范、质量检查、日志与配置
- 测试与 CI：单元测试、回归基线、数据质量门禁
- 交接：运行手册、培训、Q&A

**交付物**
- C1：pipeline 工程包（脚本/配置/README）
- C2：测试与 CI 配置
- C3：运行手册 + 培训材料

**验收**
- 客户在其环境中复现运行；完成交接清单
