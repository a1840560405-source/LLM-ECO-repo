# 贡献与复现指南

感谢你对本项目的关心与贡献！为保证研究透明与可复现，请遵循以下流程：

## 提交问题（Issues）

- 若发现文档错误、表述不清或需要补充的案例，请先在 Issues 中提问并附上下文。 

## 贡献流程（Pull Request）

1. Fork 仓库并在 feature 分支上修改。
2. 提交 PR 时请在描述中包含：变更摘要、变更理由、是否会影响可复现性、测试步骤。
3. 若新增数据或语料，请说明许可/来源并确保已脱敏或取得使用许可。

## 数据与隐私

- 禁止上传包含个人敏感信息的原始文本（身份证号、完整联系方式等）。
- 若因研究需要分享受限数据，请提供合成示例或脱敏后的代表性样本，并在 README 或数据说明中注明获取方式与许可。

## 可复现性要求

- 若提交关于指标构建或校正流程的代码（示例脚本、Jupyter Notebook），请附带最小可运行示例与示例数据（或生成示例数据的脚本）。
- 对于微调/微调数据切片，请说明时间窗、数据筛选规则与任何预处理步骤。

## 引用与学术诚信

- 若使用本白皮书内容，请引用 `CITATION.md` 中给出的参考格式。

## 联系方式

对于大型协作（算力共享、数据集合作），请通过 Issues 或在仓库主页留言以便安排进一步沟通。
